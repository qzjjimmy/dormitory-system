# 学生宿舍管理系统 — 功能文档

## 系统概述

学生宿舍管理系统是一套基于 Spring Boot + Vue 3 + MySQL 的 Web 应用，支持学生、管理员、宿管员三种角色，覆盖宿舍日常运营的主要场景：入住管理、报修维修、费用缴纳、访客预约、调宿申请、卫生检查、晚归登记、即时通讯、AI 智能问答，以及**基于多特征兼容性评分的智能宿舍分配算法**。

- **后端**：Spring Boot 2.7 + JDBC + WebSocket + MySQL
- **前端**：Vue 3 + Vite + 原生 CSS
- **AI**：可选接入 DeepSeek API

## 功能模块详述

### 1. 登录认证

- 支持三个角色快速登录，登录页提供角色快捷按钮
- 账号体系：`admin` / `student` / `dormkeeper`，默认密码均为 `123456`
- **BCrypt 密码加密**：用户密码使用 BCrypt 哈希存储，旧明文密码首次登录时自动升级
- **Token 鉴权**：登录成功后返回 UUID token，后续请求通过 `Authorization: Bearer <token>` 头携带
- 后端 `LoginInterceptor` 拦截所有 `/api/**` 请求，未授权访问返回 401
- 登录状态持久化到 `sessionStorage`

### 2. 学生端首页

- 个性化欢迎语 + 北京时间实时时钟 + 天气信息
- 安全提醒 Hero Banner
- 校园公告列表
- 快捷入口卡片：我的宿舍、账户余额、报修申请
- 卫生评分圆环图表（本月综合得分 + 全楼排名 + 上次检查结果）
- 室友列表

### 3. 我的宿舍

- 宿舍基本信息：楼栋、房号、房型、床位
- 室友列表（含在寝/离寝状态，高亮本人）
- **床位布局可视化**：鸟瞰图展示宿舍平面结构（大门、四床、阳台），标注室友姓名
- **设施运行状态**：照明、空调、网络、直饮水四类设备实时状态（正常/运行中/滤芯待换）
- **卫生月度指标**：评分圆环 + 全楼排名 + 上次检查结果

### 4. 报修管理

- 学生提交报修申请（类型、位置、描述）
- 管理员/宿管查看、编辑、删除报修记录
- 状态流转：待处理 → 处理中 → 已完成 / 已驳回
- **一键审核**：管理员/宿管可直接点击"通过"或"驳回"快速处理
- 支持关键字搜索

### 5. 费用管理

- 水电费账单生成与记录
- 缴费状态：待缴费 / 已缴费
- 金额统计汇总

### 6. 访客预约

- 默认展示预约记录表格（访客姓名、来访时间、事由、审核状态）
- 点击"新增预约"按钮展开表单
- 状态：待审核 / 已通过 / 已拒绝

### 7. 服务评价

- 综合评分圆环（优秀/良好/合格/待改进四级）
- 历史评价记录列表
- 提交评价表单

### 8. 调宿申请

- 默认展示历史申请表格（期望宿舍、原因、审核状态）
- 点击按钮展开申请表，显示当前宿舍信息
- 状态：待审核 / 审核中

### 9. 新生注册与特征采集

- 登录页底部"立即注册"入口，两页表单：账号信息 + 10 项个人特征
- 特征项：性别、专业班级、作息习惯、是否抽烟、兴趣爱好、卫生习惯、是否打游戏、是否打鼾、返回宿舍时间、噪音容忍度
- 房间偏好：可选 4人间/6人间 + 床位偏好（靠窗/靠门/无所谓）
- 注册后自动登录，未分配宿舍前菜单受限
- 老用户首次登录时强制补全特征弹窗

### 10. 智能宿舍分配算法

- **两阶段贪心匹配**：阶段一同班优先按兼容性评分分配，阶段二跨班补位
- **兼容性评分模型**：性别硬约束 + 10 维特征加权（作息25/抽烟20/卫生15/游戏10/打鼾10/返回时间10/噪音5/兴趣5）
- **4/6人间支持**：算法根据学生偏好创建不同容量宿舍，同类型满了才跨类型分配
- **调宿推荐**：计算与目标宿舍现有成员的兼容度，返回 Top-3 推荐
- 管理员在"智能分配"页面一键运行，显示分配结果矩阵（含兼容度均分），确认后写入数据库

**涉及文件**：`RoomAssignmentService.java`、`AssignmentController.java`、`SmartAssignment.vue`、`RegisterForm.vue`、`ProfileCompletionModal.vue`

### 11. AI 智能助手

- 对接 **DeepSeek API**（deepseek-chat 模型），真实 AI 对话
- 预设系统提示词限定为宿舍事务领域（报修、缴费、访客、调宿、卫生、晚归等）
- **多轮对话**：自动保存对话历史到 `localStorage`，支持新建/切换/删除对话
- 左侧对话历史列表、中间消息气泡流、右侧智能建议面板
- API 调用失败自动降级为本地回复，保证演示可用
- 预设热门咨询快捷入口

### 12. 实时天气

- 后端 `WeatherController` 对接 wttr.in 免费 API，实时获取天气数据
- 前端首页顶部展示：城市、天气状况、温度、湿度、风速
- 无需登录即可访问，API 不可用时降级显示默认值

### 13. 智能通话（即时通讯）

- 联系人列表（同宿舍楼室友 > 宿管员 > 其他，自动排序）
- **WebSocket 实时推送**：发送消息后通过 WebSocket 推送到在线接收方
- 未读消息红点角标 + 自动刷新
- 消息自动标记已读

### 14. 入住管理（管理员/宿管）

- 在住人员统计卡片（在住人数、登记记录数）
- 入住人员卡片网格（姓名、宿舍、床位、在住/离寝状态）
- 办理入住/退宿表单

### 15. 房间管理（管理员/宿管）

- 房源统计（房间总数、已住满、可入住）
- 房间卡片网格展示（房号、楼栋、配置描述、状态标签）
- 新增/编辑/删除房间

### 16. 卫生检查（宿管）

- 宿舍卫生打分记录
- 评分：优秀 / 良好 / 合格 / 待改进

### 17. 晚归登记（宿管）

- 晚归学生姓名、时间、原因登记

### 18. 物品出入登记（宿管）

- 大件物品搬入/搬出登记（物品名称、型号、审批状态）

### 19. 意见反馈

- 学生提交意见建议
- 管理员/宿管查看、回复

## 技术要点

- **BCrypt 密码加密**：用户密码使用 `BCryptPasswordEncoder` 哈希存储，旧明文密码首次登录时自动升级
- **Token 鉴权**：`LoginInterceptor` 统一拦截验证，`TokenStore` 内存管理 session
- **幂等种子数据**：项目启动时自动建表并插入演示数据，所有种子数据（用户/业务记录/聊天消息）均通过 `COUNT=0` 判断避免重复插入
- **分页查询**：`RecordService.list()` 和 `UserService.list()` 均已接入 `LIMIT ? OFFSET ?` 分页，返回 `{ rows, total, page, size, pages }`
- **参数校验**：所有 Controller 入口均做参数空值校验，`AuthController` 拒绝空用户名/密码
- **单元测试**：`AuthControllerTest` 覆盖 7 个测试用例（登录/CRUD/鉴权）
- **WebSocket 实时推送**：聊天消息通过 WebSocket handler 的 `pushToUser()` 方法实时推送到在线用户
- **对话持久化**：AI 对话历史按用户 ID 隔离存储到 `localStorage`，切换账号互不可见
- **智能宿舍分配算法**：两阶段贪心匹配，10 维特征加权兼容性评分，支持 4/6人间偏好，含热力图可视化和调宿推荐
- **审核工作流**：管理员/宿管端一键通过/驳回待审核记录
- **WebSocket 实时推送**：聊天消息实时推送，宿舍分配完成后自动通知学生刷新
- **修改密码**：账户设置支持修改登录密码（旧密码验证）
- **响应式布局**：CSS 适配 1400px / 1280px / 820px 三个断点

## 启动方式

**前置条件**：JDK 8+、Maven 3.6+、Node.js 18+、MySQL 8.0

方式一：双击 `start.bat`，自动启动后端和前端。

方式二：手动启动
```bash
# 后端
mvn spring-boot:run

# 前端（新终端）
cd frontend
npm install       # 首次运行
npm run dev
```

- 后端：http://localhost:8080
- 前端：http://localhost:5173
